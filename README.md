# PowerSimulator

中国古代官场题材的浏览器静态 Demo。玩家扮演一名入仕士人，在十二年仕途中通过卡牌构筑处理公事、经营关系、积累名声，并在皇权、士林、地方治理、政敌和个人压力之间塑造自己的为官方法。

当前项目可直接打开 `index.html` 运行，无需构建流程。代码使用浏览器全局对象 `GameData` / `Game` 组织，存档写入 `localStorage`。

## 运行方式

1. 直接用浏览器打开 `index.html`。
2. 如需本地服务，可在仓库根目录启动任意静态文件服务器，例如：

```bash
python -m http.server 8000
```

然后访问 `http://localhost:8000/`。

## 已实现内容

- 12 年仕途流程：从 24 岁入仕开始，按春、夏、秋、冬四季推进。
- 3 个官职阶段：翰林、知县、御史，根据官评 `merit` 达到阈值后自动升迁。
- 87 张卡牌：覆盖奏疏、清议、公文、人情、法度、政务、财政、权谋、家族、人物、休养、才学、污点、心病等类型。
- 27 张卡牌拥有分支用法，共 54 个出牌模式；部分模式会设置连招前置、提高威胁或触发反制。
- 38 个事件：18 个官职基础事件、8 个关系触发事件、12 个具名 NPC 事件。
- 8 个关系节点：皇帝、座师、同年、地方士绅、胥吏、上司、政敌、士林。
- 12 名具名 NPC：每人有传记、议程、触发条件，以及羁绊、信任、亏欠、怨恨等状态。
- 13 个结局：包含强制旧案成狱，以及清节、能臣、权臣、仁政、酷吏、贪腐、党祸、退隐、滞留馆阁、地方仕途、圆滑善终和普通结局。
- 4 个主视图：当前事务、关系朝局、卡组牌库、生平卷。
- 基础操作：抽牌、出牌、结束本季、保留 1 张手牌、弃 2 张手牌换资源、存档、重开。
- 结算系统：结案总结弹窗、事件后果、关系变动、NPC 牵连、自动成长、压力入心病、污点入库、升迁记录。
- 生平系统：记录仕途履历、事件摘要、人物评价和结局，并支持导出生平文本。
- 翰林基础牌平衡：新增《同年转圜》处理 `政敌反扑`，并让《托付同年》《官样文章》分别补足 `举荐不足`、`证据不足` 的替代解法。

## 核心玩法循环

1. 当前官职与关系状态决定本季事件。
2. 玩家抽取手牌，并用行动卡降低事件轨道风险或改变世界、名声、关系和资源。
3. 每张卡会消耗精力、银两、人情或压力容量，并可能带来污点、心病、怨恨或后续牵连。
4. 结束本季后结算事件，生成结案总结、记录生平，并根据结果发放自动成长或负面后果。
5. 时间推进到下一季，关系事件、NPC 事件、升迁、强制结局和最终结局继续根据状态触发。

## 主要视图

- 当前事务：展示人物状态、当前事件、事件轨道、手牌、朝局指标和生平日志。
- 关系朝局：查看各关系节点、具名 NPC、风险标签和关系动态。
- 卡组牌库：查看抽牌堆、手牌、弃牌堆、保留牌与卡牌详情。
- 生平卷：展示最终结局、生平摘要、仕途履历、人物评价和导出入口。

## 代码结构

src/data/endings.js        结局数据
assets/                    UI mockup 与切片素材
tmp/previews/              本地预览截图
bureaucracy_ck3_deckbuilder_design.md  详细玩法设计文档
