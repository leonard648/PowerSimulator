# PowerSimulator

万历新政题材的浏览器静态 Demo。玩家固定扮演张居正，从万历元年奉遗诏辅政开始，在十年首辅之局中维持帝心、推动考成与财赋新政、处理内廷牵连、言路反扑、部院执行、地方清丈阻力和身后清算风险。

项目可直接打开 `index.html` 运行，无需构建流程。代码使用浏览器全局对象 `GameData` / `Game` 组织，存档写入 `localStorage`。新版使用独立存档 key，旧虚构官场存档不会混入万历设定。

## 运行方式

1. 直接用浏览器打开 `index.html`。
2. 如需本地服务，可在仓库根目录启动任意静态文件服务器，例如：

```bash
python -m http.server 8000
```

然后访问 `http://localhost:8000/`。

## 已实现内容

- 10 年万历时间线：从万历元年（1573）推进到万历十年，按春、夏、秋、冬四季结算。
- 3 个执政阶段：辅政初定、新政推行、权势晚期。内部仍沿用 `hanlin` / `county` / `censor` ID 以降低重写风险。
- 固定主角：张居正，字叔大，湖广江陵人，开局身份为内阁首辅、帝师。
- 史实 NPC：万历帝、李太后、冯保、高拱、张四维、申时行、王国光、潘季驯、戚继光、吴中行、邹元标、张敬修。
- 事件池覆盖：遗诏辅政、高拱去位余波、经筵训主、考成法、清丈田亩、一条鞭法、治河、边防、夺情、言官弹劾、帝心转冷、身后清算等。
- 手段库成长：票拟、经筵、考成、联络内廷、会商阁臣、清丈财赋、河工、边镇、言路压制、帝心维系和身后保全。
- 生平系统：记录新政履历、事件摘要、人物牵连、结局评价，并支持导出《张居正万历新政纪略》。

## 核心玩法循环

1. 当前万历年份、执政阶段、关系状态决定本季事务。
2. 玩家依次选择铺垫/查明与定案/表态处置，压低事件轨道风险。
3. 处置会消耗精力、银两、人情或增加压力，并改变名声、朝局、关系和 NPC 状态。
4. 本季结束后生成结算弹窗，记录余波，并沉淀新的新政手段或污点、心病。
5. 年份推进会按万历时间线切换阶段，最终在万历十年进入张居正的结局判定。

## 代码结构

```text
index.html          页面结构和脚本入口
styles.css          全局样式与响应式布局

src/core/state.js   初始状态、存档、时间线阶段归一化
src/core/deck.js    手段库、阶段手段包、污点和心病入口
src/core/rules.js   事件处置、结算、关系/NPC 触发和结局
src/ui/render.js    所有视图、弹窗、生平卷渲染
src/data/cards.js   手段数据
src/data/events.js  事件与两阶段处置数据
src/data/offices.js 时间线、关系节点和 NPC 数据
src/data/endings.js 结局数据
```
